# 📚 Glosario de Nerve Community

Bienvenido al glosario. Aquí explicamos los términos más comunes que usamos en el desarrollo, Git y GitHub, para que cualquiera pueda entenderlos sin importar su nivel de experiencia.

| Término (Inglés/Español) | 🔍 ¿Qué es? | 🎯 ¿Para qué sirve? | 💻 Ejemplo de uso |
| :--- | :--- | :--- | :--- |
| **Repositorio (Repo)** | Carpeta de proyecto | Guarda todos los archivos de tu proyecto y el historial de todos los cambios que se han hecho. | `https://github.com/Kaia-Alenia/nerve-community` |
| **Fork** | Copia personal | Crea una copia de un proyecto ajeno en tu cuenta de GitHub para que puedas modificarlo sin afectar el original. | Clic en el botón "Fork" en la página de GitHub. |
| **git clone** | Descarga del código | Trae el repositorio desde internet a tu computadora para que puedas editar los archivos. | `git clone https://github.com/TU_USUARIO/nerve-community.git` |
| **Commit** | Punto de guardado | Guarda los cambios que hiciste en los archivos, adjuntando un pequeño mensaje de lo que modificaste. | `git commit -m "Agrega solución al reto"` |
| **Push** | Subir cambios | Envía tus *commits* (cambios guardados localmente) hacia tu repositorio en GitHub. | `git push origin main` |
| **Pull** | Actualizar / Bajar | Descarga los cambios más recientes que alguien más haya subido a GitHub hacia tu computadora. | `git pull origin main` |
| **Branch (Rama)** | Línea de trabajo paralela | Permite hacer pruebas o crear nuevas funciones sin romper el código principal (la rama `main`). | `git checkout -b solucion-reto-01` |
| **Pull Request (PR)** | Petición de integración | Solicitas a los creadores del proyecto que revisen tus cambios y los agreguen al proyecto oficial. | Crear un PR en GitHub después de hacer *push*. |
| **Merge** | Fusión de código | Acción de aceptar un *Pull Request* y unir tu código nuevo con el proyecto principal. | "¡Felicidades, hicimos merge de tu PR!" |
| **Issue** | Ticket / Tarea | Un foro dentro de GitHub para reportar errores, pedir funciones nuevas o listar tareas pendientes. | Crear un "Issue" para avisar que algo no funciona. |
| **Bug** | Error de software | Un fallo en el código que hace que el programa no funcione como se esperaba o se cierre inesperadamente. | "Encontré un bug en la calculadora de IMC." |

> 💡 **IMPORTANTE:** Estos términos son la base para empezar a contribuir en cualquier proyecto Open Source. ¡No te preocupes si no los memorizas todos de golpe, los dominarás con la práctica! 🚀
